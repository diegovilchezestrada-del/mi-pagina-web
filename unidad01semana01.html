📘 Informe Técnico Comparativo sobre Arquitecturas de Bases de Datos

1. Introducción

Las arquitecturas de bases de datos definen cómo se organizan, almacenan, acceden y gestionan los datos en un sistema. En la actualidad, con el crecimiento de aplicaciones web y móviles en entornos educativos, es fundamental elegir la arquitectura adecuada considerando escalabilidad, disponibilidad, seguridad y costo.

En este informe se analizan tres arquitecturas principales: Centralizada, Cliente-Servidor y Distribuida.

2. Definición y Características

🔹 Arquitectura Centralizada

Todos los datos residen en un único servidor.

  • Un solo punto de control.
  • Fácil administración.
  • Alta consistencia de datos.
  • Dependencia total del servidor.

🔹 Arquitectura Cliente-Servidor

Los clientes solicitan servicios a un servidor central.

  • Separación entre cliente y servidor.
  • Mayor escalabilidad.
  • Uso en aplicaciones web modernas.
  • Control de seguridad centralizado.

🔹 Arquitectura Distribuida

Los datos se almacenan en múltiples nodos conectados.

  • Alta disponibilidad.
  • Escalabilidad horizontal.
  • Tolerancia a fallos.
  • Mayor complejidad.

3. Ventajas y Desventajas

Arquitectura Ventajas Desventajas
Centralizada Fácil de administrar, bajo costo Punto único de fallo
Cliente-Servidor Buen rendimiento, escalable Dependencia del servidor
Distribuida Alta disponibilidad, escalable Complejidad y costo alto

4. Ejemplos Actuales

  • Centralizada: SQL Server en servidor local.
  • Cliente-Servidor: MySQL, PostgreSQL con apps web.
  • Distribuida: Cassandra, Google Spanner, CockroachDB.

5. Recomendación

Se recomienda la arquitectura Cliente-Servidor para sistemas académicos debido a su equilibrio entre rendimiento, seguridad y facilidad de implementación.

6. Conclusión

Cada arquitectura responde a necesidades distintas. La centralizada es simple, la cliente-servidor es equilibrada y la distribuida es altamente escalable. Para entornos educativos, la mejor opción es cliente-servidor.
